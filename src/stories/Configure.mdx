# Trinity Storybook

Storybook is the source of truth for Trinity UI components.

## What this Storybook covers

- Visual states and behavior for reusable components.
- Interactive docs generated from `autodocs`.
- Browser-based story tests through Vitest + Playwright.
- Accessibility checks via `@storybook/addon-a11y`.

## Component taxonomy

- **Atoms**: primitives like `Button`, `Input`, `Logo`.
- **Molecules**: compositions like `VideoCard`, `Dropdown`, `ContentLink`.
- **Organisms**: larger structures like `Navbar`.
- **Templates**: layout-level components.

## Commands

```bash
npm run storybook
npm run build-storybook
npx vitest run
```

## Story quality bar

- Include realistic props/content (Portuguese content where appropriate).
- Add key states (default, edge, loading/empty/auth where relevant).
- Use decorators only when needed to represent app context.
- Add `tags: ['autodocs']` for component stories.

## Publishing

Chromatic is the primary target for hosted Storybook docs in this project.
